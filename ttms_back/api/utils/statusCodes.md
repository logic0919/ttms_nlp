# HTTP 状态码使用规范

## 常用状态码说明

### 2xx 成功类

- **200 OK** - 请求成功，最常用的成功状态码
  - 用于：GET、POST、PUT、DELETE 等操作成功
  - 示例：登录成功、注册成功、查询成功

- **201 Created** - 资源创建成功
  - 用于：创建新资源（如新建用户、新建订单）
  - 示例：用户注册成功

- **204 No Content** - 成功但无返回内容
  - 用于：删除操作成功、更新操作成功但不需要返回数据

### 4xx 客户端错误类

- **400 Bad Request** - 请求参数错误
  - 用于：参数格式错误、缺少必需参数、业务逻辑错误
  - 示例：密码格式错误、账号已被占用、验证失败

- **401 Unauthorized** - 未授权/认证失败
  - 用于：未登录、token无效、token过期
  - 示例：未提供token、token验证失败

- **403 Forbidden** - 禁止访问
  - 用于：已登录但无权限访问
  - 示例：普通用户访问管理员接口

- **404 Not Found** - 资源不存在
  - 用于：请求的资源不存在
  - 示例：用户不存在、订单不存在

- **409 Conflict** - 资源冲突
  - 用于：资源已存在、重复操作
  - 示例：账号已被占用、重复提交

- **422 Unprocessable Entity** - 请求格式正确但无法处理
  - 用于：数据验证失败
  - 示例：数据格式正确但不符合业务规则

### 5xx 服务器错误类

- **500 Internal Server Error** - 服务器内部错误
  - 用于：服务器未知错误、数据库错误、代码异常
  - 示例：数据库连接失败、未捕获的异常

- **503 Service Unavailable** - 服务不可用
  - 用于：服务暂时不可用
  - 示例：服务器维护中、数据库维护中

## 本项目推荐使用的状态码

### 最常用的状态码（90%的场景）

1. **200** - 成功
2. **400** - 请求错误（参数错误、业务逻辑错误）
3. **401** - 认证失败（未登录、token无效）
4. **404** - 资源不存在
5. **500** - 服务器错误

### 使用示例

```javascript
// 成功
res.status(200).send(success(data, '操作成功'))

// 参数错误
res.status(400).send(error('参数格式错误', 400))

// 认证失败
res.status(401).send(error('请提供token', 401))

// 资源不存在
res.status(404).send(error('用户不存在', 404))

// 服务器错误
res.status(500).send(error('服务器内部错误', 500))
```

## 响应格式

所有响应统一格式：

```javascript
// 成功响应
{
    "status": 200,
    "code": 200,
    "message": "操作成功",
    "data": { ... }
}

// 错误响应
{
    "status": 400,
    "code": 400,
    "message": "错误信息"
}
```
